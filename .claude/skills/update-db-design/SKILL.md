# DB設計書更新スキル

実装したマイグレーションに基づいて、DB設計書を自動更新するためのスキルです。

## 前提条件

### 必須ファイル

以下のファイルが存在する必要があります：

- `docs/database-design.md` - DB設計書
- `src/database/migrations/` - Laravelマイグレーションファイル
- `src/database/seeders/` - シーダーファイル

### 既存設計書の優先順位

`docs/database-design.md` に既存の設計がある場合：

1. **既存のDB設計書** - 最優先
   - プロジェクトの実装に基づいている
   - このスキルのテンプレートより優先する

2. **このスキルのテンプレート** - 参考資料
   - 新しいテーブル追加時の参考
   - 既存パターンに合わせて調整

## 更新対象

このスキルで更新できる項目：

### 1. 新しいテーブル追加
- テーブル名、カラム定義、データ型
- 制約（NOT NULL, UNIQUE, DEFAULT など）
- インデックス設定

### 2. テーブルへのカラム追加
- 既存テーブルへの新しいカラム定義
- マイグレーションに対応する設計書更新

### 3. インデックス定義追加
- 複合インデックス
- ユニークインデックス
- パフォーマンス最適化用インデックス

### 4. リレーション定義更新
- 外部キー制約
- カスケード削除設定
- リレーション図の更新

## 実装の手順

このスキルの詳細な使用方法は以下を参照してください：

- **テンプレート**: `./template.md`
- **詳細ガイド**: `./guide.md`

## 出力先

更新内容は以下に保存されます：

```
docs/database-design.md        # DB設計書（メイン）
src/database/migrations/       # 実装マイグレーション（参考）
```

## 使用例

新しい取引レポート機能を実装した場合：

```
実装内容:
- 新しいテーブル作成: reports, report_items
- 新しいカラム追加: transactions に report_id
- 新しいインデックス: transactions(user_id, report_id)

DB設計書更新:
- テーブル定義追加: reports, report_items
- カラム追加: transactions.report_id
- ER図更新
- インデックス戦略更新
- リレーション定義更新
```

## 注意事項

- ✅ 既存テーブル定義に合わせて新規追加
- ✅ マイグレーションファイルの内容を正確に反映
- ✅ リレーション（外部キー）を正しく定義
- ✅ インデックス戦略を効率的に設計
- ❌ 手動で実装したマイグレーションを修正しない
- ❌ 既存テーブルの定義を誤解しない（マイグレーション履歴を確認）

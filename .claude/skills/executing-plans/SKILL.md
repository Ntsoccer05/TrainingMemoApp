# 計画の実行

## 概要

計画を読み込み、批判的にレビューし、全タスクを実行し、完了したら報告します。

**開始時に宣言してください:** 「executing-plans スキルを使って実装を進めます。」

**補足:** サブエージェントが利用できる場合は、代わりに `superpowers:subagent-driven-development` を使うと品質が大幅に向上します。

## プロセス

### ステップ1: 計画の読み込みとレビュー
1. 計画ファイルを読む
2. 批判的にレビュー — 疑問点や懸念事項を特定する
3. 懸念がある場合: 開始前にユーザーに確認する
4. 懸念がない場合: TodoWrite を作成して進める

### ステップ2: タスクの実行

各タスクについて：
1. `in_progress` にマーク
2. 各ステップを正確に実行（計画は細かいステップに分解済み）
3. 指定された検証を実行
4. `completed` にマーク

**コミットは各タスクでは行わない。** 計画にコミットステップが含まれていてもスキップし、ステップ3でまとめてコミットする。

### ステップ3: 開発の完了

全タスク完了・検証後：
1. すべてのテストがパスすることを確認する
2. 全変更をまとめてコミットする

```bash
git add -A
git commit -m "feat: <実装内容を一言で>"
```

3. 作業完了をユーザーに報告し、マージ・PR・ブランチ保持などの次のアクションはユーザーが判断する

## 停止してサポートを求めるタイミング

**以下の場合は即座に実行を停止してください：**
- ブロッカーに遭遇した（依存関係の欠如、テスト失敗、指示が不明確）
- 計画に実装を始められない重大なギャップがある
- 指示を理解できない
- 検証が繰り返し失敗する

**推測するのではなく、確認を求めてください。**

## 前のステップに戻るタイミング

**レビュー（ステップ1）に戻る場合：**
- ユーザーがフィードバックに基づいて計画を更新した
- 根本的なアプローチの再検討が必要

**ブロッカーを無理に突破しないでください** — 停止して確認を取ってください。

## 注意事項
- まず計画を批判的にレビューする
- 計画のステップを正確に従う
- 検証をスキップしない
- 計画がスキルを参照している場合はそれを使う
- ブロックされたら停止し、推測しない
- ユーザーの明示的な同意なしに main/master ブランチで実装を開始しない

## 連携

**必須ワークフロースキル:**
- **superpowers:using-git-worktrees** — 隔離されたワークスペースを確保
- **superpowers:writing-plans** — このスキルが実行する計画を作成

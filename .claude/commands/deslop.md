# AI コードスロップの除去

デフォルトブランチとの差分を確認し、このブランチで導入されたAI生成の不要コードをすべて除去します。

## ワークフロー

### ステップ1: ベースブランチの決定

以下の優先順で適切なベースブランチを選ぶ:

1. `origin/main`（存在する場合）
2. `origin/master`（存在する場合）
3. `main`（ローカル）
4. `master`（ローカル）

以降のステップでは `<base>` としてこのブランチを使用する。

### ステップ2: 差分の確認

- 変更ファイル一覧を表示:
  - `git diff --name-only <base>...HEAD`
- 全差分を表示:
  - `git diff <base>...HEAD`

### ステップ3: AI生成スロップの除去

変更されたファイルを編集して、意図した動作を保ちながらスロップを除去する。

除去対象:
- 人間なら書かないコメント、またはファイルの他の箇所と不整合なコメント
- そのコードエリアでは異常な過剰な防御チェックや try/catch ブロック（信頼済み・検証済みのコードパスから呼ばれる場合は特に）
- 不要な間接参照・ラッパー・過度な抽象化、明確な価値なく導入されたノイジーなヘルパー関数
- ファイルやプロジェクトの規約と不整合なその他のスタイル

ガイドライン:
- 変更範囲を最小限に: スロップ除去に必要な箇所のみ触る
- 不要な場合はコードの書き直しや整形をしない
- 新しい抽象化を追加するより不要なコードを削除することを優先する
- 意図が不明な場合は周囲のコードから推測し、動作を互換に保つ

### ステップ4: 最終レポート

最後に **1〜3文**の変更サマリーのみを報告する。
